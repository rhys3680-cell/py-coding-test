# 베이지안 통합 관점

## 1. 배경 / 왜 나왔나

ELO / Glicko / IRT를 각각 따로 배우면 **공식 3개를 외우는 느낌**이 됨. 하지만 셋 다 **같은 통계적 뼈대**에서 파생된 특수 케이스임을 보면:

- 왜 Glicko가 ELO보다 "자연스러운 일반화"인지
- 왜 IRT가 "문제 버전 ELO"처럼 생겼는지
- 어디를 수정하면 **내 학습 데이터에 맞는 커스텀 모델**을 만들 수 있는지

가 한눈에 보임. 이 문서는 **수식 하나(베이즈 정리) + 하나의 관찰(로지스틱 링크)**로 세 모델을 재해석.

## 2. 핵심 아이디어

**"레이팅은 점수가 아니라 실력에 대한 확률 분포다."**

- **믿음(belief) = 확률분포** p(θ)
- **새 관측** = 정/오답 또는 승/패 u
- **업데이트** = 베이즈 정리:

```
p(θ | u) ∝ p(u | θ) × p(θ)
```

| 용어 | 의미 |
|---|---|
| p(θ) | **사전분포 (prior)** — 현재 내가 믿는 실력 |
| p(u | θ) | **우도 (likelihood)** — 실력이 θ라면 관측 u가 나올 확률 |
| p(θ | u) | **사후분포 (posterior)** — 관측 후 갱신된 믿음 |

ELO/Glicko/IRT는 **p(θ), p(u|θ)를 어떻게 잡고, 업데이트를 어떻게 근사하느냐**의 차이일 뿐.

## 3. 세 모델의 공통 구조

### 3.1 공통 우도: 로지스틱 링크

셋 다 "승률/정답률"을 **로지스틱 함수**로 모델링:

```
P(u=1 | θ, φ) = σ(스케일 × (θ - φ))
```

- θ = 내 실력
- φ = 상대 실력(ELO/Glicko) 또는 문제 난이도(IRT)
- 스케일 = ELO의 ln10/400, IRT 2PL의 a, ...

**다른 점은 φ에 뭘 넣느냐 + θ가 분포인가 점인가**.

### 3.2 세 모델의 prior/posterior 선택

| 모델 | p(θ) 형태 | 업데이트 방식 | φ는 무엇 |
|---|---|---|---|
| **ELO** | 점 추정 R (분산 없음) | 점 그대로 이동 (Δ = K×(S-E)) | 상대 R_B (점 추정) |
| **Glicko** | 정규분포 N(R, RD²) | 평균·분산 모두 업데이트 | 상대 N(R_B, RD_B²) |
| **Glicko-2** | 정규 N(R, RD²) + 변동성 σ | 평균·분산·변동성 업데이트 | 같음 |
| **IRT 2PL** | 정규 N(0, 1) 가정 (MMLE) | 문제 풀이 응답에서 MLE/MAP | 문제 b(점 또는 prior) |

→ **Glicko는 θ에 정규 prior를 붙인 ELO**. **IRT는 φ를 문제로 바꾼 다인 ELO**.

### 3.3 왜 "정규분포 × 로지스틱"이 편한가

- 로지스틱 우도와 정규 prior를 곱하면 **닫힌 형태 사후분포가 없음** (conjugate가 아님).
- 그래서 **라플라스 근사** (사후를 MAP 근처 정규로 근사)가 자주 쓰임.
- Glicko의 업데이트 공식 = 라플라스 근사의 결과.

```
posterior ≈ N(MAP, (관측 Fisher 정보 + prior 정밀도)^-1)
```

- 이게 바로 Glicko의 `1/RD_new² = 1/RD² + 1/d²` 형태. **정밀도(precision) 덧셈 법칙**.

## 4. 통일된 업데이트 공식

정규 prior `θ ~ N(μ, σ²)`에 로지스틱 우도 n개가 들어올 때 라플라스 근사:

```
1/σ²_new = 1/σ² + I_obs
μ_new    = μ + σ²_new × S_obs
```

- `I_obs = Σ (스케일)² × P_i × (1-P_i)` — 관측 Fisher 정보 (=Glicko의 1/d²)
- `S_obs = Σ (스케일) × (u_i - P_i)` — 관측 스코어 (=Glicko의 Σ g × (s-E))

**이 두 줄이 Glicko 본체**. ELO는 여기서 "prior 분산 무시, K로 근사"한 형태, IRT MAP 추정은 여기서 "φ를 문제 파라미터로 치환"한 형태.

## 5. 모델 간 대응표

| 개념 | ELO | Glicko | IRT 2PL |
|---|---|---|---|
| 실력 | R (점) | (R, RD) | θ ~ N(0,1) |
| 상대/문제 | R_B (점) | (R_B, RD_B) | (a, b) |
| 우도 | σ(β(R_A-R_B)) | σ(β × g(RD_B) × (R_A-R_B)) | σ(a(θ-b)) |
| 업데이트 | Δ=K(S-E) | prec-add | MLE/MAP |
| 시간 변화 | 없음 | c로 RD 증가 | 기본 없음 (Dynamic IRT로 추가) |
| 팀/다인 | 없음 | 없음 | 없음 (→ TrueSkill) |

## 6. 숫자로 확인: 같은 데이터, 세 모델

**상황**: 내가 3경기/문제를 겪음.
- 상대/문제 1: 난이도 0, 승/정답
- 상대/문제 2: 난이도 0, 승/정답
- 상대/문제 3: 난이도 1, 패/오답

(비교 단순화 위해 스케일 통일: σ(θ-b), K와 a=1 고정, 초기 θ=0.)

### ELO (K=1 — 과장 없이 보려고)
- E_1 = 0.5, Δ = 1-0.5 = +0.5 → θ=0.5
- E_2 = σ(0.5) ≈ 0.622, Δ = 1-0.622 = +0.378 → θ ≈ 0.878
- E_3 = σ(0.878-1) = σ(-0.122) ≈ 0.470, Δ = 0-0.470 = -0.470 → θ ≈ 0.408

### IRT 2PL MLE (정적)
- 앞서 `03_irt.md` 4.2 예제와 동일 구조 → MLE θ ≈ **1.0** 근처

### Glicko (라플라스 근사, prior N(0, 1))
- 각 관측의 Fisher 정보 = p(1-p) ≈ 0.25 × 3 = 0.75
- 새 정밀도 = 1 + 0.75 = 1.75 → 새 분산 ≈ 0.571 → 새 RD ≈ 0.756
- S_obs ≈ (1-0.5) + (1-σ(?)) + ... 를 MAP 반복 계산하면 θ ≈ **0.7 근처**

**관찰**:
- ELO는 관측을 순서대로 소비해 "최근 결과에 끌림" (0.408로 저평가).
- IRT MLE는 batch 전체를 한 번에 보고 정답 2/3의 평균적 추정 (1.0).
- Glicko/베이지안은 prior N(0,1)이 아래로 당겨서 IRT와 ELO 중간(0.7).

→ **세 모델은 "같은 데이터에 다른 추정 정책"을 적용할 뿐**.

## 7. 이 관점이 주는 실전 이점

### 7.1 커스텀 모델 설계가 쉬워진다
- "내 학습 데이터에 맞는 모델"을 만들고 싶다면:
  1. prior p(θ) 정하기 (정규? 두꺼운 꼬리?)
  2. 우도 p(u|θ) 정하기 (로지스틱? 풀이시간까지 포함? 부분점수?)
  3. 베이즈 업데이트 (닫힌 형태면 analytical, 아니면 라플라스/VI/MCMC)
- ELO/Glicko/IRT 중 뭘 "고를지"가 아니라 **내 데이터에 맞게 조립**하는 관점.

### 7.2 "풀이시간"을 우도에 넣기
- 맞췄다/틀렸다(이진)만 쓰지 말고 **풀이시간 t**도 관측에 추가:
  ```
  P(u, t | θ) = P(u|θ) × P(t|θ, u)
  ```
- 시간 모델은 로그정규 등이 흔함. 이러면 "빨리 맞춘 것"과 "겨우 맞춘 것"을 구분.

### 7.3 유형별 θ의 계층 모델
- 전체 θ를 가지고, 유형별 θ_DP, θ_그래프를 **계층(prior as shared parent)**으로 묶기.
- 데이터 적은 유형은 전체 평균으로 **shrinkage**, 많으면 개별 추정.
- Bayesian hierarchical model — PyMC/Stan으로 구현 가능.

### 7.4 실험 설계에 직접 연결
- 초기 단계(**prior 넓음**) → 탐색 위주 (다양한 유형)
- 수렴(**prior 좁음**) → 약점 유형에 Fisher 정보 많은 문제 집중
- 오래 쉼(**prior 분산 인위적 증가**) → 재측정 우선

## 8. 한계 / 주의점

1. **로지스틱이 만능은 아님**
   - 문제-사람 상호작용이 복잡하면(유형 편향, 힌트 효과 등) 로지스틱 링크는 단순. 신경망 기반 DKT가 대안.
2. **라플라스 근사는 분포가 단봉일 때만**
   - 다봉/비대칭 posterior에선 MCMC/VI 필요.
3. **표본 부족 시 prior 지배**
   - 한 달 쉬고 한 문제만 풀면 사후는 prior에 거의 같음 → 추정값을 과신 금지.
4. **IID 가정 위반**
   - 같은 유형 연속 풀면 상관 있음. 계층 모델 또는 KT 필요.
5. **계산량**
   - 베이지안 관점을 **그대로** 구현하면 무거움. Glicko 같은 라플라스 근사가 현실적 타협점.
