## 정성 관찰 — 반복된 마찰 테마

### 1. **"파이썬 전용 메서드 모름 → 루프로 우회"** (가장 빈번)
- 181950: `"abc" * 3` 반복 연산자 대신 for 루프
- 181949: `swapcase()` 대신 `islower/upper/lower` 분기
- 181946: `print(..., sep="")` 대신 `print` 2번 호출
- 181945: `"\n".join()` 대신 for 루프

→ **문법이 아니라 어휘력 문제**. JS보다 파이썬 표준 라이브러리가 풍부한데, 그 레퍼토리를 아직 모름.

### 2. **슬라이스 vs 인덱스 혼동** (181943)
- `my_string[s+len(x)]` (IndexError) vs `my_string[s+len(x):]` (안전)
- **슬라이스는 범위 초과 안전, 인덱스는 에러** — 파이썬 고유 특성.

### 3. **입력 파싱 관례**
- `split(' ')` 대신 `split()`이 안전·관례
- `.strip()`은 대부분 불필요

### 4. **변수명에 내장 타입 사용**
- `str = input()`, `str1, str2 = ...`
- 내장을 가려서 같은 스코프 내 `str(...)` 호출 불가 위험.

## 학습 포인트 (본인이 이미 쓰고 있는 파이써닉 관용구)
- `map(int, input().split())` — 완전 정착
- 튜플 언패킹 — 자연스러움
- f-string + 표현식 평가 — 자연스러움
- 삼항 `A if cond else B` — 자연스러움 (181937에서 바로 사용)

## 해석

- **병목은 문법 전환이 아니라 어휘 부족.** JS 사고로 풀 수 있으면 코드는 나오지만, **"더 짧은 파이썬 표현"**이 무엇인지 모름.
- Lv.0은 로직이 단순해서 **이 병목이 드러날 여지가 적음**. 같은 레벨을 반복해도 학습효과 포화.
- 다음 단계 필요: **Lv.1 진입**으로 자료구조/내장모듈(collections, itertools 등) 쓸 상황을 강제로 만들기.
